# 🚀 Roadmap

**Binary to Blog** প্রোগ্রামে আপনাকে স্বাগতম। আগামী ৪ সপ্তাহে আপনি কেবল কম্পিউটার ব্যবহার করা নয়, বরং কম্পিউটারকে ‘কমান্ড’ করা শিখবেন। আমরা শুরু করব Manjaro Linux-এর খুঁটিনাটি দিয়ে এবং শেষ করব আপনার নিজের একটি গ্লোবাল পোর্টফোলিও সাইট + ব্লগ তৈরির মাধ্যমে। সেখানে আপনি লেখালেখি, পোস্ট শেয়ার ও নিজের personal journal হিসেবে ব্যবহার করতে পারবেন। দেখতে সেটা অনেকটা এরকম হবে,
-  <https://tanvirislm.github.io/>

## 🛠 Checklist

সপ্তাহ ১ শুরু করার আগে নিশ্চিত করুন আপনার কাছে নিচের বিষয়গুলো আছে,

- [x] **Manjaro KDE Plasma** বা যেকোনো একটা লিনাক্স distro ইনস্টল করা আছে।
- [x] একটি অ্যান্ড্রয়েড বা আইওএস (iOS) ডিভাইস (নেটওয়ার্কিং টাস্কের জন্য)।
- [x] একটি **GitHub** অ্যাকাউন্ট।
- [ ] হাতে ফ্রী সময়, প্রতিদিন কিছু শেখার এবং ভুল হলে তা সমাধান করার মানসিকতা। প্রতিদিন হয়তো ১-২ ঘন্টা সময় যেতে পারে।

## 📅 সপ্তাহ ১: কোর এনভায়রনমেন্ট এবং ডকুমেন্টেশন

**লক্ষ্য:** টার্মিনাল এবং প্লেইন-টেক্সট ডকুমেন্টেশনে দক্ষতা অর্জন করা।

### 🔍 আলোচনার বিষয়

- [ ] **সিস্টেম ম্যানেজমেন্ট:** `pacman`, `yay` (AUR), এবং মিরর সেটআপ।
    
- [ ] **ইকোসিস্টেম কানেক্টিভিটি:** **KDE Connect** সেটআপ করা (ফোন এবং পিসির মেলবন্ধন)। [**অপশনাল**]
    
- [ ] **মার্কডাউন (Markdown):** সুন্দর করে গুছিয়ে কন্টেন্ট লেখা।
    
- [ ] **শেল (Shell):** নেভিগেশন এবং ফাইল ম্যানিপুলেশন।
    

### 📅 প্রতিদিনের গাইড

- [ ] **দিন ১:** সিস্টেম আপডেট এবং KDE Connect। মিরর আপডেট, ফুল সিস্টেম আপগ্রেড এবং ল্যাপটপের সাথে ফোনের **KDE Connect** পেয়ারিং।
    
- [ ] **দিন ২:** AUR এবং টুলস। `yay` ইনস্টল করা এবং একটি টেক্সট এডিটর (VS Code/Obsidian) ইনস্টল করা।
    
- [ ] **দিন ৩:** মার্কডাউন বেসিক। হেডার, লিস্ট এবং লিঙ্ক তৈরি করে `cheatsheet.md` ফাইল তৈরি করা।
    
- [ ] **দিন ৪:** অ্যাডভান্সড মার্কডাউন। সিনট্যাক্স হাইলাইটিং সহ কোড ব্লক এবং টেবিল তৈরি প্র্যাকটিস।
    
- [ ] **দিন ৫:** নেভিগেশন মাস্টারি। মাউস ব্যবহার না করে `cd`, `ls -la`, এবং `tree` ব্যবহার করা শিখুন।
    
- [ ] **দিন ৬:** ফাইল অপারেশন। `cp`, `mv`, `mkdir`, এবং `rm` প্র্যাকটিস করুন।
    
- [ ] **দিন ৭:** সপ্তাহ ১ রিভিউ। আপনার `week1_logs.md` ফাইলটি সম্পূর্ণ করুন।
    

### 🏠 সপ্তাহ ১-এর হোমওয়ার্ক

- [ ] **অ্যাপ ইনস্টলেশন:** টার্মিনাল ব্যবহার করে আপনার নিত্যপ্রয়োজনীয় ৩টি অ্যাপ (যেমন: VLC, Discord, বা Inkscape) ইনস্টল করুন।
    
- [ ] **লগ ফাইল:** একটি `setup_log.md` ফাইল তৈরি করুন যেখানে আপনার লিনাক্স ইন্সটল করার সময় করা প্রতিটি প্রধান কমান্ডের তালিকা থাকবে।


## 📅 সপ্তাহ ২: নেটওয়ার্কিং এবং ডেভেলপার ওয়ার্কফ্লো

**লক্ষ্য:** প্রফেশনালদের মতো প্রগ্রেস 'সেভ' করা এবং রিমোটলি কম্পিউটার ব্যবহার করা।

### 🔍 আলোচনার বিষয়

- [ ] **গিট (Git) বেসিক:** কমিট (Commit) লাইফসাইকেল।
    
- [ ] **SSH:** সিকিউর কি (Keys) এবং রিমোট টার্মিনাল অ্যাক্সেস।
    
- [ ] **টার্মিনাল মাল্টিপ্লেক্সিং:** **Zellij** দিয়ে সেশন ম্যানেজমেন্ট। [**অপশনাল**]
    
- [ ] **ডটফাইলস (Dotfiles):** কনফিগারেশন ফাইল ট্র্যাক করা।
    

### 📅 প্রতিদিনের গাইড

- [ ] **দিন ৮:** গিট পরিচিতি। নোটস ফোল্ডারে `git init` করে প্রথম কমিট করা।
    
- [ ] **দিন ৯:** SSH আইডেন্টিটি। Ed25519 কি-পেয়ার জেনারেট করে গিটহাবে যুক্ত করা।
    
- [ ] **দিন ১০:** রিমোট পোর্টাল। `openssh` ব্যবহার করে অন্য ডিভাইস থেকে ল্যাপটপে SSH কানেক্ট করা।
    
- [ ] **দিন ১১:** Zellij/Tmux। টার্মিনাল স্ক্রিন স্প্লিট করা এবং সেশন ম্যানেজমেন্ট। [**অপশনাল**]
    
- [ ] **দিন ১২:** ব্রাঞ্চিং। গিটে নতুন ব্রাঞ্চ তৈরি করা এবং মার্জ করা।
    
- [ ] **দিন ১৩:** মার্জ কনফ্লিক্ট। ইচ্ছাকৃতভাবে কনফ্লিক্ট তৈরি করে তা সমাধান করা।
    
- [ ] **দিন ১৪:** ডটফাইলস। কনফিগারেশন ফাইলগুলো একটি নির্দিষ্ট রিপোজিটরিতে ব্যাকআপ রাখা।
    

### 🏠 সপ্তাহ ২-এর হোমওয়ার্ক

- [ ] **গিটহাব পুশ:** গিটহাবে একটি নতুন রিপোজিটরি তৈরি করে আপনার মার্কডাউন নোটগুলো সেখানে `push` করুন।
    
- [ ] **রিমোট কমান্ড:** ফোন থেকে SSH দিয়ে কানেক্ট করে ল্যাপটপে একটি নতুন ফোল্ডার তৈরি করুন এবং ল্যাপটপটি রিমোটলি শাটডাউন (Shutdown) করার কমান্ড দিন।
    

## 📅 সপ্তাহ ৩: স্ট্যাটিক সাইট আর্কিটেকচার (Hugo)

**লক্ষ্য:** শুরু থেকে একটি হাই-পারফরম্যান্স ওয়েবসাইট তৈরি করা।

### 🔍 আলোচনার বিষয়

- [ ] **হুগো (Hugo) বেসিক:** সাইট জেনারেশন এবং স্ট্রাকচার।
    
- [ ] **থিমিং:** গিট সাব-মডিউল (Git Submodules)।
    
- [ ] **অ্যাসেট অপ্টিমাইজেশন:** টার্মিনাল ব্যবহার করে ইমেজ কম্প্রেস করা।
    
- [ ] **লোকাল ডেভেলপমেন্ট:** হুগো লাইভ সার্ভার।
    

### 📅 প্রতিদিনের গাইড

- [ ] **দিন ১৫:** হুগো ইনস্টল। `hugo new site portfolio` দিয়ে নতুন প্রোজেক্ট শুরু।
    
- [ ] **দিন ১৬:** থিম সেটআপ। গিট সাব-মডিউল হিসেবে পছন্দের থিম যুক্ত করা।
    
- [ ] **দিন ১৭:** কনফিগারেশন। `hugo.toml` ফাইলে নিজের ব্যক্তিগত তথ্য যুক্ত করা।
    
- [ ] **দিন ১৮:** প্রথম কন্টেন্ট। "About Me" পেজ এবং প্রথম ব্লগ পোস্ট লেখা।
    
- [ ] **দিন ১৯:** শর্টকোড। মার্কডাউনে ইমেজ এবং ইউটিউব ভিডিও এমবেড করা।
    
- [ ] **দিন ২০:** ইমেজ অপ্টিমাইজেশন। `imagemagick` দিয়ে ব্লগের ছবি কম্প্রেস করা। [**অপশনাল**]
    
- [ ] **দিন ২১:** লেআউট টুইকস। থিমের বেসিক কালার বা ফন্ট পরিবর্তন করার চেষ্টা।
    

### 🏠 সপ্তাহ ৩-এর হোমওয়ার্ক

- [ ] **কন্টেন্ট ক্রিয়েশন:** আপনার সাইটে অন্তত ৩টি আলাদা ব্লগ পোস্ট যুক্ত করুন (যেমন: লিনাক্স কেন শিখছেন, আপনার প্রিয় কমান্ড ইত্যাদি)।
    
- [ ] **ইমেজ চ্যালেঞ্জ:** আপনার ব্লগে ব্যবহার করা প্রতিটি ছবির সাইজ ৫০০ কেবি (KB)-এর নিচে নিয়ে আসুন। [**অপশনাল**]
    
- [ ] **কাস্টমাইজেশন:** সাইটের ফুটার (Footer) বা মেনুবারে আপনার সোশ্যাল মিডিয়া লিঙ্কগুলো যুক্ত করুন।
    

## 📅 সপ্তাহ ৪: ডিপ্লয়মেন্ট এবং অটোমেশন (CI/CD)

**লক্ষ্য:** আপনার কাজকে ইন্টারনেটে পাবলিশ করা এবং অটোমেশন শেখা।

### 🔍 আলোচনার বিষয়

- [ ] **GitHub Actions:** অটোমেটেড বিল্ড পাইপলাইন।
    
- [ ] **হোস্টিং:** স্ট্যাটিক হোস্টিং প্রোভাইডার।
    
- [ ] **DNS এবং SEO:** সাইটকে গুগলে খুঁজে পাওয়ার উপযোগী করা। [**অপশনাল**]
    

### 📅 প্রতিদিনের গাইড

- [ ] **দিন ২২:** হোস্টিং সেটআপ। গিটহাবে সাইটের কোড পুশ করা।
    
- [ ] **দিন ২৩:** ম্যানুয়াল ডিপ্লয়। Netlify বা GitHub Pages-এ সাইট লাইভ করা।
    
- [ ] **দিন ২৪:** অটোমেশন (CI)। GitHub Actions সেটআপ করা যাতে পুশ করলেই সাইট আপডেট হয়।
    
- [ ] **দিন ২৫:** SEO চেক। মেটা ট্যাগ এবং সাইট ম্যাপ নিয়ে কাজ করা। [**অপশনাল**]
    
- [ ] **দিন ২৬:** কাস্টমাইজেশন। ফেভিকন এবং চূড়ান্ত স্টাইলিং।
    
- [ ] **দিন ২৭:** গ্র্যাজুয়েশন পোস্ট। পুরো মাসের অভিজ্ঞতার ওপর একটি পোস্ট লেখা।
    
- [ ] **দিন ২৮:** গ্র্যাজুয়েশন ডে! লাইভ লিঙ্ক সবার সাথে শেয়ার করা।
    

### 🏠 সপ্তাহ ৪-এর হোমওয়ার্ক

- [ ] **লাইভ লিঙ্ক:** আপনার তৈরি করা ওয়েবসাইটের লিঙ্কটি আপনার মেন্টরকে পাঠান।
    
- [ ] **পারফরম্যান্স টেস্ট:** Google PageSpeed Insights ব্যবহার করে আপনার সাইটের স্কোর চেক করুন এবং এটি ৯০-এর উপরে রাখার চেষ্টা করুন।
    
- [ ] **ফাইনাল ব্লগ:** একটি চূড়ান্ত ব্লগ পোস্ট লিখুন যার শিরোনাম হবে "Binary to Blog: My 30-Day Transformation"।
